# ADR-003: Политика секретов (ENV, без хардкода)
Дата: 2025-10-20
Статус: Accepted

## Context
Нужны ключи (например, к внешнему Calendar API). Их нельзя хардкодить в код/репозиторий и логи.

## Decision
Секреты читаем из переменных окружения (`CAL_API_KEY` и др.). Приложение не стартует при отсутствии ключа (fail fast). Логи/ответы не содержат секреты.

## Consequences
+ Исключаем утечку секретов через VCS.
+ Готовность к последующей ротации (интеграция с Vault/KMS позднее).
+ Требуется настройка окружения для запусков/CI.
